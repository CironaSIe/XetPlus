# XET+ 配置参考

本文档列出 `xet config` 命令支持的所有配置选项。

## 📋 配置优先级

XET+ 使用多层配置系统，优先级从高到低：

1. **命令行参数** - 如 `--token`, `--proxy`
2. **环境变量** - 如 `XET_TOKEN`, `HTTPS_PROXY`
3. **项目配置** - `./.xet/config.toml`
4. **用户配置** - `~/.xetrc`
5. **系统配置** - `/etc/xet/config.toml`

## 🔧 可配置项

### [xet] - XET 核心设置

#### `xet.endpoint`
- **说明**: CAS 服务器地址
- **默认值**: `https://cas.xethub.com`
- **环境变量**: `XET_ENDPOINT`
- **用法**:
  ```bash
  xet config xet.endpoint https://cas.xethub.com
  ```

#### `xet.token`
- **说明**: HuggingFace 认证 Token（必需）
- **默认值**: 无
- **环境变量**: `XET_TOKEN`
- **用法**:
  ```bash
  xet config xet.token hf_xxxxxxxxxxxxxxxxxxxxx
  ```
- **注意**: Token 可从 https://huggingface.co/settings/tokens 获取

---

### [network] - 网络设置

#### `network.hf_endpoint`
- **说明**: HuggingFace API 端点
- **默认值**: `https://huggingface.co`
- **环境变量**: `HF_ENDPOINT`
- **用法**:
  ```bash
  # 使用国内镜像（推荐）
  xet config network.hf_endpoint https://hf-mirror.com
  
  # 恢复默认
  xet config network.hf_endpoint https://huggingface.co
  ```
- **常用值**:
  - `https://huggingface.co` - 官方端点（国外）
  - `https://hf-mirror.com` - 国内镜像（快速）

#### `network.proxy`
- **说明**: HTTP/HTTPS 代理地址
- **默认值**: 无
- **环境变量**: `HTTPS_PROXY`, `https_proxy`
- **用法**:
  ```bash
  # 设置代理
  xet config network.proxy http://127.0.0.1:7890
  
  # 或使用 SOCKS5 代理
  xet config network.proxy socks5://127.0.0.1:1080
  
  # 清除代理
  xet config --unset network.proxy
  ```
- **注意**: 
  - 支持 `http://`, `https://`, `socks5://` 协议
  - 代理不影响已优选的直连域名（如 cdn.xethub.com）

#### `network.optimize_hosts`
- **说明**: 是否启用 HOST IP 优选
- **默认值**: `false`
- **环境变量**: `XET_OPTIMIZE_HOSTS`
- **用法**:
  ```bash
  # 启用（推荐国内网络）
  xet config network.optimize_hosts true
  
  # 禁用
  xet config network.optimize_hosts false
  ```
- **效果**: 
  - 启用后自动测试多个 IP，选择最快的
  - 国内网络速度可提升 3-10x
  - 优选的域名自动跳过代理

---

### [cache] - 缓存设置

#### `cache.dir`
- **说明**: Xorb 磁盘缓存目录
- **默认值**: `~/.xet/cache`
- **环境变量**: `XET_CACHE_DIR`
- **用法**:
  ```bash
  # 自定义缓存目录
  xet config cache.dir /data/cache/xet
  
  # 使用默认目录
  xet config --unset cache.dir
  ```
- **注意**: 
  - 确保目录有写入权限
  - 缓存可重复使用，加速下载 36x
  - 使用 `--keep-cache` 保留下载后的缓存

---

### [download] - 下载设置

#### `download.concurrency`
- **说明**: 并行下载的默认并发数
- **默认值**: `4`
- **环境变量**: `XET_CONCURRENCY`
- **用法**:
  ```bash
  # 提高并发（网络好时）
  xet config download.concurrency 16
  
  # 降低并发（避免限流）
  xet config download.concurrency 2
  ```
- **建议值**:
  - 国外网络/代理: 4-8
  - 国内直连: 8-16
  - 网络不稳定: 2-4

---

### [logging] - 日志设置

#### `logging.level`
- **说明**: 日志输出级别
- **默认值**: `INFO`
- **环境变量**: `XET_LOG_LEVEL`
- **用法**:
  ```bash
  # 详细调试
  xet config logging.level DEBUG
  
  # 正常输出
  xet config logging.level INFO
  
  # 只显示警告和错误
  xet config logging.level WARNING
  ```
- **可选值**: `DEBUG`, `INFO`, `WARNING`, `ERROR`

---

## 📝 配置文件格式

XET+ 使用 TOML 格式的配置文件（`~/.xetrc`）：

```toml
[xet]
token = "hf_xxxxxxxxxxxxxxxxxxxxx"
endpoint = "https://cas.xethub.com"

[network]
hf_endpoint = "https://hf-mirror.com"
proxy = "http://127.0.0.1:7890"
optimize_hosts = true

[cache]
dir = "/data/cache/xet"

[download]
concurrency = 16

[logging]
level = "INFO"
```

## 🛠️ 配置管理命令

### 查看所有可用配置项（推荐）
```bash
xet config --list-all
```
显示所有可用配置项的完整信息：配置键、说明、默认值、当前值、环境变量。

### 查看当前配置
```bash
xet config --list
```
只显示已设置的配置项。

### 获取单个配置值
```bash
xet config --get xet.token
xet config --get network.proxy
```

### 设置配置
```bash
xet config <key> <value>
```

### 删除配置
```bash
xet config --unset <key>
```

## 🌐 国内网络优化方案

### 方案 1: 使用 hf-mirror（推荐）
```bash
xet config network.hf_endpoint https://hf-mirror.com
```
- ✅ 无需代理
- ✅ 速度快
- ✅ 完整支持 XET 协议

### 方案 2: 通过代理访问
```bash
xet config network.proxy http://127.0.0.1:7890
```
- ✅ 访问官方源
- ⚠️  需要本地代理工具

### 方案 3: 启用 HOST 优选
```bash
xet config network.optimize_hosts true
```
- ✅ 自动选择最快 IP
- ✅ 优选域名跳过代理
- ⚠️  首次下载需要测速（约 5-10 秒）

### 组合使用（最佳）
```bash
# 使用镜像 + 优选（双重加速）
xet config network.hf_endpoint https://hf-mirror.com
xet config network.optimize_hosts true

# 或：代理 + 优选
xet config network.proxy http://127.0.0.1:7890
xet config network.optimize_hosts true
```

## 🔍 配置示例

### 极简配置（国内用户）
```bash
# 只需两步
xet config xet.token hf_xxxxxxxxxxxxxxxxxxxxx
xet config network.hf_endpoint https://hf-mirror.com
```

### 完整配置（高级用户）
```bash
# 认证
xet config xet.token hf_xxxxxxxxxxxxxxxxxxxxx

# 网络优化
xet config network.hf_endpoint https://hf-mirror.com
xet config network.optimize_hosts true
xet config download.concurrency 16

# 自定义缓存
xet config cache.dir /data/cache/xet

# 调试
xet config logging.level DEBUG
```

### 代理环境配置
```bash
# Token
xet config xet.token hf_xxxxxxxxxxxxxxxxxxxxx

# 代理
xet config network.proxy http://127.0.0.1:7890

# 优选（让 CAS 直连）
xet config network.optimize_hosts true
```

## 💡 提示

1. **环境变量覆盖配置文件**
   ```bash
   export XET_TOKEN=hf_xxxx
   # 临时覆盖 ~/.xetrc 中的设置
   ```

2. **命令行参数优先级最高**
   ```bash
   xet download user/repo/file.gguf --proxy http://127.0.0.1:7890
   # 即使配置文件中有代理设置，也会使用命令行指定的
   ```

3. **查看实际生效的配置**
   ```bash
   xet config --list
   # 显示合并后的最终配置（包括环境变量）
   ```

4. **重置为默认**
   ```bash
   # 删除用户配置文件
   rm ~/.xetrc
   ```

## 🆘 常见问题

### Q: 如何查看所有可用的配置项？
A: 使用 `xet config --list-all` 查看完整的配置项列表，包括说明、默认值、当前值和对应的环境变量。

### Q: `--list` 和 `--list-all` 有什么区别？
A: 
- `--list`: 只显示已设置的配置
- `--list-all`: 显示所有可用配置项（包括未设置的），带完整说明

### Q: 配置文件在哪里？
A: 
- 用户配置: `~/.xetrc`
- 项目配置: `./.xet/config.toml`（可选）
- 系统配置: `/etc/xet/config.toml`（可选）

### Q: 如何验证配置是否生效？
A: 使用 `xet config --list` 查看，或在下载时观察日志输出。

### Q: 是否必须设置所有选项？
A: 不需要。只有 `xet.token` 是必需的，其他都有合理的默认值。
