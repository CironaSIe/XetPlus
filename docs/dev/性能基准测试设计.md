## 📋 概述

为 XET+ 设计系统化的性能基准测试框架，量化各项优化的实际效果，为未来优化提供数据支撑。

---

## 🎯 测试维度

### 1. Xorb 缓存效果测试

**目标**: 量化磁盘缓存的加速效果

**测试场景**:
- 冷启动下载（无缓存，首次下载）
- 热启动下载（完全命中缓存）
- 部分命中场景（共享 xorb 的不同文件）
- 缓存清理后重新下载

**测量指标**:
- 下载总时间（秒）
- 网络流量（MB）
- 缓存命中率（%）
- 加速比（冷启动 / 热启动）

**预期结果**:
- 完全命中缓存：加速 30-40x
- 部分命中：加速 3-10x

---

### 2. HOST 优选效果测试

**目标**: 评估 HOST 优选在不同网络环境下的效果

**测试场景**:
- 无优选 vs 有优选（国内网络）
- 无优选 vs 有优选（国外网络）
- DoH 查询耗时
- 优选结果的持久化效果（cache_ttl）

**测量指标**:
- 下载速度（MB/s）
- DoH 查询耗时（ms）
- 优选开销（首次 vs 后续）
- 加速比

**预期结果**:
- 国内网络：加速 3-10x
- 国外网络：加速 1-2x 或无明显效果
- DoH 查询：< 500ms

---

### 3. 并行写入性能测试

**目标**: 验证并行批量写入的性能提升

**测试场景**:
- 顺序写入 vs 并行写入
- 不同文件大小（100MB / 1GB / 5GB）
- GlobalWriter vs SegmentedWriter

**测量指标**:
- 写入总时间（秒）
- 内存峰值（MB）
- 磁盘 I/O 吞吐量（MB/s）
- 加速比

**预期结果**:
- 大文件（5GB+）：加速 2-3x
- 中等文件（1GB）：加速 1.5-2x
- 小文件（<500MB）：无明显加速或略慢（开销）

---

### 4. RetryCoordinator 开销测试

**目标**: 测量全局重试协调的性能开销

**测试场景**:
- 正常场景（无重试）
- 低失败率场景（10% 失败）
- 高失败率场景（50% 失败）
- 全局停止触发

**测量指标**:
- 协调器开销（相对无协调器的时间增加）
- 重试次数
- 全局停止触发时机

**预期结果**:
- 正常场景：开销 < 5%
- 高失败率：正确触发全局停止

---

### 5. 自适应并发控制（ACC）效果

**目标**: 验证 ACC 的自动调优能力

**测试场景**:
- 固定并发数（4 / 8 / 16 / 32）
- ACC 自动调整
- 不同网络条件（快速 / 慢速 / 不稳定）

**测量指标**:
- 下载吞吐量（MB/s）
- 并发数变化曲线
- 收敛速度（达到最优并发的时间）
- 成功率

**预期结果**:
- ACC 自动找到最优并发数
- 收敛时间 < 30 秒
- 最终吞吐量接近固定最优并发

---

### 6. 与 xet.py 基准对比

**目标**: 对比新旧实现的性能差异

**测试场景**:
- 相同文件、相同网络条件
- 带缓存 vs 不带缓存
- 启用所有优化 vs 禁用所有优化

**测量指标**:
- 下载时间
- 内存占用（峰值 / 平均）
- 代码复杂度（行数、模块数、圈复杂度）

**预期结果**:
- 性能持平或更好
- 内存占用相近
- 代码复杂度更低（模块化架构）

## 📊 指标收集

### 装饰器模式

```python
from benchmark.utils.metrics_collector import measure_time, measure_memory

@measure_time
@measure_memory
def download_with_cache(url: str):
    # 下载逻辑
    pass
```

### 上下文管理器模式

```python
from benchmark.utils.metrics_collector import MetricsCollector

with MetricsCollector() as mc:
    download_file(url)
    
    print(f"Time: {mc.elapsed}s")
    print(f"Peak memory: {mc.peak_memory_mb}MB")
    print(f"Network bytes: {mc.network_bytes}")
```

### 收集的指标

- **时间指标**:
  - 总下载时间
  - Xorb 下载时间
  - 组装时间
  - DoH 查询时间
  
- **网络指标**:
  - 总流量（bytes）
  - 缓存节省流量（bytes）
  - 平均速度（MB/s）
  
- **资源指标**:
  - 内存峰值（MB）
  - 内存平均值（MB）
  - 磁盘 I/O（读/写次数）
  
- **重试指标**:
  - 重试次数
  - 成功率（%）
  - 全局停止触发次数

## 🚀 使用方式

### 运行全部测试

```bash
cd benchmark
python run_benchmark.py --all
```

### 运行特定测试

```bash
python run_benchmark.py --cache      # 仅缓存测试
python run_benchmark.py --host       # 仅 HOST 优选测试
python run_benchmark.py --parallel   # 仅并行写入测试
```

### 指定测试文件

```bash
python run_benchmark.py --cache \
    --file "mykor/granite-embedding-97m-multilingual-r2-GGUF/model.gguf"
```

### 输出格式

```bash
python run_benchmark.py --all --format json  # JSON 输出
python run_benchmark.py --all --format md    # Markdown 输出（默认）
python run_benchmark.py --all --format both  # 两者都输出
```

## 📝 注意事项

### 测试前准备

1. 清空 xorb 缓存：`rm -rf ~/.cache/xet/xorbs/`
2. 确保网络稳定
3. 关闭其他大流量应用
4. 确保有足够磁盘空间（20GB+）

### 测试环境要求

- Python 3.8+
- 稳定网络连接
- 足够磁盘空间（测试文件 + 缓存）
- 建议在相同网络环境下多次测试取平均值

### 测试结果解读

- **加速比 < 1.0x**: 可能存在性能退化，需要调查
- **加速比 1.0x - 1.5x**: 效果不明显，可能需要优化
- **加速比 > 1.5x**: 优化有效
