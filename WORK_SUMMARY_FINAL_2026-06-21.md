## 🐛 问题修复

### 1. XET 文件检测问题

**用户报告**: "请你仔细检查下刚才说不是XET文件的那个，我只能说真的是，这个判定是错误的"

**根本原因**: `detect_xet_file()` 硬编码使用 `main` 分支，无法处理特定 commit 上的文件

**修复方案**:
- 添加 `--revision/-r` 参数支持
- 更新 `detect_xet_file()` 函数签名
- 更新所有调用点

**提交**: f3e5538

### 2. 自动 Fallback 功能

**用户建议**: "能否默认main不存在时候，采用自动探测最新的commit？"

**实现方案**:
- 当 revision='main' 返回 404 时
- 自动调用 HuggingFace API 获取最新 commit
- 递归调用使用最新 commit 重试

**提交**: e5da598

## 💡 关键技术点

### 1. HuggingFace API 探索
发现了 3 种获取最新 commit 的方法：
- `/api/models/{repo_id}` → 返回 sha
- `/api/models/{repo_id}/revision/{branch}` → 分支 sha
- HEAD 请求 `X-Repo-Commit` header

### 2. 递归 Fallback 设计
```python
if resp.status_code == 404 and revision == "main":
    latest_sha = api_get_sha(repo_id)
    return detect_xet_file(..., revision=latest_sha)
```
