# 修改计划

## 总览

| # | 问题 | 文件 | 改动类型 | 优先级 |
|---|------|------|---------|--------|
| 1 | Queue 承载 bytes → 2GB 峰值内存 | `segmented_reconstructor.py` | 改 queue 语义 | P0 |
| 2 | Recon 缓存错位（V1 fallback 不走共享缓存） | `cas_client.py` | 改 V1 fallback 路径 | P1 |
| 3 | 顺序写 `.part` 无 fsync | `chunk_assembler.py` | 新增周期性 fsync | P1 |
| 4 | 预取在同步下载后触发，时机偏晚 | `chunk_assembler.py` | 交换预取 & 下载顺序 | P1 |
| 5 | 模式② .tmp → copyfileobj 二次 I/O | `segmented_reconstructor.py` + `chunk_assembler.py` | 支持偏移写入 | P1 |

## 1. Queue 消息从 bytes 改为 tmp_path

### 当前设计

```
_process_segment_parallel()                  _writer_worker()
┌──────────────────────────┐                ┌──────────────────────┐
│ stream_assembly_to_temp  │                │ while True:          │
│   → .tmp (磁盘)          │  queue.put     │   item = queue.get() │
│ → read_bytes()           │───────────────→│   offset,data,idx   │
│ → (offset, bytes, idx)   │  (256MB/格)    │   f.write(data)      │
└──────────────────────────┘                │   mark_completed()   │
                                            └──────────────────────┘
```

**问题**: Queue 每格 256MB，`maxsize=2×parallel` → 8 格 = 2GB。

### 目标设计

```
_process_segment_parallel()                  _writer_worker()
┌──────────────────────────┐                ┌──────────────────────────┐
│ stream_assembly_to_temp  │  queue.put     │ while True:              │
│   → .tmp (磁盘)          │───────────────→│   item = queue.get()     │
│ → (offset, idx, tmp_path)│  (~200B/格)    │   offset,idx,path = item │
└──────────────────────────┘                │   open(path,'rb') as s  │
                                            │   copyfileobj(s, f)     │
                                            │   path.unlink()         │
                                            │   mark_completed()      │
                                            └──────────────────────────┘
```

### 具体变更

**`segmented_reconstructor.py:879-901` — `_download_and_assemble_segment`**

当前：
```python
def _download_and_assemble_segment(self, seg, recon) -> bytes:
    seg_temp = self.temp_dir / f"seg_{seg.index}.tmp"
    try:
        self._stream_assembly_to_temp(seg, recon, seg_temp)
        return seg_temp.read_bytes()
    finally:
        if seg_temp.exists():
            seg_temp.unlink()
```

改为：
```python
def _download_and_assemble_segment(self, seg, recon) -> Path:
    seg_temp = self.temp_dir / f"seg_{seg.index}.tmp"
    self._stream_assembly_to_temp(seg, recon, seg_temp)
    return seg_temp
```

清理延迟到 writer 消费后，worker 失败时残留由 `_cleanup_temp_files` 兜底。

**`segmented_reconstructor.py:677-688` — `_process_segment_parallel`**

当前：
```python
segment_data = self._download_and_assemble_segment(seg, recon)
if self._stop_event.is_set():
    return
self._write_queue.put((seg.start, segment_data, seg.index))
```

改为：
```python
seg_temp = self._download_and_assemble_segment(seg, recon)
if self._stop_event.is_set():
    seg_temp.unlink(missing_ok=True)
    return
self._write_queue.put((seg.start, seg.index, seg_temp))
```

**`segmented_reconstructor.py:575-647` — `_writer_worker`**

当前 unpacking + f.write(data)（line 605-616）：
```python
offset, data, seg_index = item
f.seek(offset)
f.write(data)
f.flush()
os.fsync(f.fileno())
self._total_written += len(data)
self._mark_segment_completed_by_index(seg_index)
```

改为：
```python
offset, seg_index, seg_temp = item
f.seek(offset)
with open(seg_temp, 'rb') as src:
    shutil.copyfileobj(src, f)
f.flush()
os.fsync(f.fileno())
seg_temp.unlink(missing_ok=True)
self._total_written += (seg_temp.stat().st_size if seg_temp.exists() else 0)
self._mark_segment_completed_by_index(seg_index)
```

**注意**: `total_written` 更新需要从实际文件大小获得。更稳健的方法是在 writer 打开文件后读取大小，但为避免在 `.tmp` 文件刚写入时 stat 出错，可以直接使用之前保存的大小信息。临时方案：在 queue message 中加入 size 字段 `(offset, seg_index, seg_temp, size)`，或者在 writer 中对 seg_temp 做 stat。

## 2. Reconstruction 缓存错位

### 当前设计

`get_segment_reconstruction`（`cas_client.py:293-313`）V1 fallback：

```python
url_v1 = f"{self.endpoint}/v1/reconstructions/{file_hash}"
resp = self.session.get(url_v1, ...)  # 无缓存，每段独立下载
full_recon = QueryReconstructionResponse.from_dict(resp.json())
return self._filter_reconstruction_by_range(full_recon, start, end)
```

**问题**: N 段 → N 次全量 recon 下载。不经过 `get_reconstruction` 的 `_recon_cache`。

### 目标设计

```python
full_recon = self.get_reconstruction(file_hash)  # 走 _recon_cache
return self._filter_reconstruction_by_range(full_recon, start, end)
```

### 具体变更

**`cas_client.py:293-313`**

当前：
```python
url_v1 = f"{self.endpoint}/v1/reconstructions/{file_hash}"
logger.debug(
    f"[CAS] 使用 V1 API（可能需要客户端过滤）: {url_v1[:80]}..."
)
resp = self.session.get(url_v1, headers=headers, timeout=self.timeout)
if resp.status_code == 401:
    self._refresh_token()
    headers = self._get_headers()
    resp = self.session.get(url_v1, headers=headers, timeout=self.timeout)
resp.raise_for_status()
full_recon = QueryReconstructionResponse.from_dict(resp.json())
return self._filter_reconstruction_by_range(full_recon, start, end)
```

改为：
```python
logger.debug("[CAS] 使用 V1 API fallback（通过 get_reconstruction 缓存）")
full_recon = self.get_reconstruction(file_hash)
return self._filter_reconstruction_by_range(full_recon, start, end)
```

**注意**: `get_reconstruction` 已经包含 token 刷新、V2/V1 协商、内存缓存写入和命中逻辑。直接用即可。第一个段的 V1 fallback 会触发一次全量 API 调用并缓存结果（TTL 1800s），后续所有段从缓存返回。

## 3. 顺序写入增加周期性 fsync

### 当前设计

`_assemble_with_sequential_write`（`chunk_assembler.py:852-982`）每个 term 写入后不做 fsync。写完直接 rename。`.part` 文件的 `confirmed_bytes` 之上可能有未落盘数据。

### 目标设计

每 N 个 term（默认 50）做一次 `f.flush()` + `os.fsync()`。N 可配置。

### 具体变更

**`chunk_assembler.py` — `__init__` 新增参数**（或类默认值）：
```python
self.fsync_interval: int = 50          # 每 N term 做一次 fsync
self._term_write_count: int = 0        # term 写入计数器
```

**`chunk_assembler.py:932-934` — term 写入后**：

当前：
```python
f.write(segment)
total_written += len(segment)
```

改为：
```python
f.write(segment)
total_written += len(segment)
self._term_write_count += 1
if self._term_write_count % self.fsync_interval == 0:
    f.flush()
    os.fsync(f.fileno())
```

**rename 后的最终 fsync**（`chunk_assembler.py:982`）：

当前：
```python
part_path.rename(output_path)
```

改为：
```python
part_path.rename(output_path)
# 可选：对目录做 fsync 保证 rename 原子性可见
# 但跨平台实现复杂，暂不做。
```

## 4. 预取在同步下载前触发

### 当前设计

两个 assembly 路径中，顺序都是：
```
1. _ensure_xorb_ready(term.hash)      # 同步下载（~10-30 秒阻塞）
2. _prefetch_upcoming_xorbs(term_idx)  # 提交预取下载
```

预取提交时当前 xorb 刚下载完，下一个 xorb 的下载从此时才开始。

### 目标设计

```
1. _prefetch_upcoming_xorbs(term_idx)  # 优先提交预取
2. _ensure_xorb_ready(term.hash)      # 同步下载当前 xorb
```

适用于两个路径：
- `_assemble_with_sequential_write`（`chunk_assembler.py:862-873`）
- `_assemble_with_prefetch`（`chunk_assembler.py:1072-1083`）

### 具体变更

**`chunk_assembler.py:862-873`** 和 **`chunk_assembler.py:1072-1083`**：

当前两段都是：
```python
self._ensure_xorb_ready(term.hash, ...)
current_cache_bytes = sum(x.memory_footprint() for x in self._xorb_cache.values())
if current_cache_bytes < low_watermark:
    self._prefetch_upcoming_xorbs(term_idx, ...)
```

统一改为：
```python
current_cache_bytes = sum(x.memory_footprint() for x in self._xorb_cache.values())
if current_cache_bytes < low_watermark:
    self._prefetch_upcoming_xorbs(term_idx, ...)
self._ensure_xorb_ready(term.hash, ...)
```

**注意**: `_prefetch_upcoming_xorbs` 内部已经检查 `_download_futures` 去重，即使 `ensure_xorb_ready` 的 xorb 被重复提交也会等待 future 而非重下载。所以交换顺序安全。

## 5. 模式② 跳过 .tmp 中转

### 当前设计

`_process_segment`（`segmented_reconstructor.py:853-867`，顺序模式）：
```
stream_assembly_to_temp → .tmp  →  copyfileobj(.tmp, output_file, offset=seg.start)  →  unlink .tmp
```

### 目标设计

让 ChunkAssembler 直接写 output 文件的目标偏移，跳过 `.tmp`。实现方式：为 `assemble_file_with_prefetch` 增加可选 `output_offset` 参数。

### 具体变更

**`chunk_assembler.py:119` — `assemble_file_with_prefetch` 签名**：

当前：
```python
def assemble_file_with_prefetch(
    self, recon, cas_client, output_path: Path, file_hash: str,
    ...
    parallel_write: bool = False,
) -> None:
```

改为：
```python
def assemble_file_with_prefetch(
    self, recon, cas_client, output_path: Path, file_hash: str,
    ...
    parallel_write: bool = False,
    output_offset: int = 0,
) -> None:
```

**`_assemble_with_sequential_write` — 打开文件时 seek**（`chunk_assembler.py:827`）：

当前：
```python
with open(part_path, file_mode) as f:
```

改为：
```python
with open(part_path, file_mode) as f:
    if output_offset > 0:
        f.seek(output_offset)
```

**`segmented_reconstructor.py:852-867` — `_process_segment`**：

当前：
```python
seg_temp = self.temp_dir / f"seg_{seg.index}.tmp"
try:
    self._stream_assembly_to_temp(seg, recon, seg_temp)
    with open(self.output_path, 'r+b') as f:
        f.seek(seg.start)
        with open(seg_temp, 'rb') as src:
            shutil.copyfileobj(src, f)
        f.flush()
        os.fsync(f.fileno())
    if seg_temp.exists():
        seg_temp.unlink()
except Exception:
    logger.info(...保留临时文件...)
    raise
```

改为：
```python
# 直接通过 ChunkAssembler 写到 output 文件的目标偏移
self._stream_assembly_to_temp(
    seg, recon, seg_temp=None,
    output_path=self.output_path, output_offset=seg.start
)
```

并且 `_stream_assembly_to_temp` 需要支持 `seg_temp=None` 时跳过 `.tmp` 组装，直接写 `output_path` 的 `output_offset`。

**`segmented_reconstructor.py:691` — `_stream_assembly_to_temp`**：

当前签名：
```python
def _stream_assembly_to_temp(self, seg, recon, seg_temp: Path) -> None:
```

改为：
```python
def _stream_assembly_to_temp(
    self, seg, recon,
    seg_temp: Optional[Path] = None,
    output_path: Optional[Path] = None,
    output_offset: int = 0,
) -> None:
```

当 `output_path` 提供时，`assemble_file_with_prefetch` 的 `output_path=output_path`、`output_offset=output_offset`，不走 `.tmp`。
当 `seg_temp` 提供时（并行模式），行为不变。

## 依赖与实施顺序

```
1 (queue 改 tmp_path) ───── 无外部依赖
2 (recon 缓存) ──────────── 无外部依赖
3 (fsync) ───────────────── 无外部依赖
4 (预取时机) ─────────────── 无外部依赖，但与 3 在同一文件
5 (模式② 直写) ───────────── 依赖 ChunkAssembler 的 output_offset 参数扩展
```

建议实施顺序：1 → 2 → 4 → 3 → 5。1 和 2 可并行。
