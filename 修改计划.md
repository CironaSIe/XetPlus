### 步骤 1：`xorb_deserializer.py` — 新增 `extract_chunks_in_range()` ✅ 已完成

> 返回 `[(local_chunk_id, chunk_data, chunk_size), ...]`。

在 `StreamingXorbAccessor` 类中（`extract_range` 方法之后，约 line 825 后）新增。

**注意**：预解压模式不能复用 `_decompress_single_entry`，因为其 `uncompressed_len=0`（构造时 line 663）。
两条路径必须分别处理，但边界查找逻辑共用（bisect）。

```python
def extract_chunks_in_range(self, start_byte: int, end_byte: int) -> list:
    """按 chunk 粒度提取范围内的数据。

    与 extract_range 不同：返回每个 chunk 的独立数据和全局 chunk_id，而非拼接后的连续字节。
    用于 per-chunk hash 计算和 xorb 补全。

    Returns:
        [(chunk_id: int, chunk_data: bytes, chunk_size: int), ...]
    """
    if start_byte < 0 or end_byte > self._data_size:
        raise ValueError(
            f"[StreamingXorb] 范围越界: [{start_byte}, {end_byte}), "
            f"data_size={self._data_size}"
        )
    if start_byte >= end_byte:
        return []

    # 通用边界查找：找 [start_byte, end_byte) 覆盖的 _index 条目
    virtual_offsets = [e.virtual_offset for e in self._index]
    start_idx = bisect.bisect_left(virtual_offsets, start_byte)
    end_idx = bisect.bisect_left(virtual_offsets, end_byte)
    if start_idx > 0:
        start_idx -= 1  # 包含 start_byte 所在的 chunk
    if end_idx >= len(self._index):
        end_idx = len(self._index)

    if self._pre_decompressed:
        # 预解压模式：从 _raw 按 chunk 边界切片
        return self._extract_chunks_predecompressed(
            start_idx, end_idx, start_byte, end_byte
        )

    # 压缩模式：逐个解压
    # _ChunkIndexEntry 无 chunk_id 字段（只有 virtual_offset, data_offset, 
    # compressed_len, comp_scheme, uncompressed_len），用 i 作为 xorb 内索引
    chunks = []
    for i in range(start_idx, end_idx):
        entry = self._index[i]
        chunk_data = self._decompress_single_entry(entry)
        local_chunk_id = i  # xorb 内从 0 开始，调用方需加 base_cid 转全局
        # 裁剪到 [start_byte, end_byte) 范围内
        chunk_start = max(0, start_byte - entry.virtual_offset)
        chunk_end = min(len(chunk_data), end_byte - entry.virtual_offset)
        if chunk_start < chunk_end:
            chunks.append((local_chunk_id,
                          chunk_data[chunk_start:chunk_end],
                          chunk_end - chunk_start))
    return chunks

def _extract_chunks_predecompressed(self, start_idx, end_idx,
                                     start_byte, end_byte) -> list:
    """预解压模式：用 _index 边界切片 _raw。"""
    # 预解压模式下 _external_offsets 存的是 global_chunk_id → virtual_offset
    # 需要反向查：从 virtual_offset → global_chunk_id
    vo_to_cid = {vo: cid for cid, vo in self._external_offsets.items()}
    chunks = []
    for i in range(start_idx, end_idx):
        entry = self._index[i]
        chunk_id = vo_to_cid.get(entry.virtual_offset, i)  # 反向查找
        # 找下一个 entry 的 virtual_offset 作为结束边界
        if i + 1 < len(self._index):
            chunk_end_vo = self._index[i + 1].virtual_offset
        else:
            chunk_end_vo = self._data_size
        # 裁剪到请求范围内
        seg_start = max(start_byte, entry.virtual_offset)
        seg_end = min(end_byte, chunk_end_vo)
        if seg_start < seg_end:
            data = self._raw[seg_start:seg_end]
            chunks.append((chunk_id, data, len(data)))
    return chunks
```

**关键变更**：返回值增加 `chunk_id`，用于跨文件补全时定位缺失的 chunk。

    # 跳过已完成的 term

**先做方案 1**（总长度校验），简单且有效。**方案 2 可选**，取决于是否需要续传安全性。

## D3 — 修复流程改为精确覆写 + repo_xorb_map

### 位置

`file_verifier.py:repair()` — 修复流程（line ~357-437）。

### 当前流程

依赖 per-term SHA256 诊断损坏位置 → 重新下载损坏的 xorb → 写入对应偏移。

**问题**：
1. per-term SHA256 是自校验（无外部锚点），无法检测"下载时就是错数据"
2. 修复后没有验证重新下载的数据是否正确

### 新流程（基于 D1 的 xorb_hash 验证）

```python
def repair_with_xorb_verification(
    self,
    output_path: str,
    file_hash: str,
    repo_xorb_map: dict,
    cas_client,
    recon: CASReconstruction
):
    """基于 xorb_hash 的精确修复流程"""
    
    # 1. 文件级 SHA256 失败 → 逐 xorb 验证
    logger.info("[FileVerifier] 开始 xorb 级诊断...")
    
    corrupted_xorbs = set()
    verified_xorbs = set()
    xorb_accumulator = {}  # {xorb_hash: [(chunk_id, chunk_hash_hex, size), ...]}
    
    for term in recon.terms:
        if term.hash in verified_xorbs:
            continue
        
        # 下载该 term 的 xorb 数据
        xorb_data = await cas_client.download_xorb(term.hash)
        
        # 提取该 term 用到的 chunk
        chunks_data = xorb_data.extract_chunks_in_range(
            term.chunks[0], term.chunks[1]
        )
        
        for chunk_id, chunk_data, chunk_size in chunks_data:
            chunk_hash_hex = merkle_hash.hash_to_hex(
                merkle_hash.compute_data_hash_bytes(chunk_data)
            )
            if term.hash not in xorb_accumulator:
                xorb_accumulator[term.hash] = []
            xorb_accumulator[term.hash].append((chunk_id, chunk_hash_hex, chunk_size))
        
        # 检查该 xorb 是否完整（需要跨文件补全）
        fetched_chunk_ids = {cid for cid, _, _ in xorb_accumulator[term.hash]}
        expected_count = repo_xorb_map.get(term.hash, {}).get("total_chunks", len(fetched_chunk_ids))
        
        if len(fetched_chunk_ids) < expected_count:
            # 跨文件补全
            await _complete_xorb_cross_file(
                term.hash, xorb_accumulator, repo_xorb_map,
                cas_client, output_path, fetched_chunk_ids
            )
        
        # 验证 xorb_hash
        chunks_list = [(h, s) for _, h, s in sorted(xorb_accumulator[term.hash])]
        computed = merkle_hash.compute_xorb_hash_from_hashes(chunks_list)
        
        if computed != term.hash:
            corrupted_xorbs.add(term.hash)
            logger.warning(f"xorb {term.hash[:16]} 验证失败")
        else:
            verified_xorbs.add(term.hash)
    
    if not corrupted_xorbs:
        logger.error("[FileVerifier] 所有 xorb 验证通过，但文件 SHA256 不匹配 — 可能是拼接错误")
        return False
    
    # 2. 修复损坏的 xorb（精确覆写）
    logger.info(f"[FileVerifier] 发现 {len(corrupted_xorbs)} 个损坏的 xorb，开始修复...")
    
    with open(output_path, 'r+b') as f:
        for xorb_hash in corrupted_xorbs:
            # 重新下载该 xorb 的所有数据（跨文件）
            xorb_info = repo_xorb_map.get(xorb_hash)
            if not xorb_info:
                logger.error(f"xorb {xorb_hash[:16]} 不在 repo_xorb_map 中，跳过")
                continue
            
            # 跨文件下载完整 xorb
            all_chunks = {}
            for other_file, term_list in xorb_info["files"].items():
                for term_info in term_list:
                    # 下载 other_file 的该 term 的 xorb 数据
                    # 解压并积累到 all_chunks
                    pass  # 实现细节见 D1.2 的 _complete_xorb_cross_file
            
            # 验证重新下载的 xorb
            chunks_list = [(h, s) for cid, h, s in sorted(all_chunks.items())]
            computed = merkle_hash.compute_xorb_hash_from_hashes(chunks_list)
            if computed != xorb_hash:
                logger.error(f"重新下载的 xorb {xorb_hash[:16]} 仍然损坏，放弃修复")
                return False
            
            # 精确覆写当前文件中用到该 xorb 的字节范围
            for term_idx, term in enumerate(recon.terms):
                if term.hash != xorb_hash:
                    continue
                
                # 计算该 term 在文件中的偏移
                file_offset = sum(
                    t.unpacked_length for t in recon.terms[:term_idx]
                )
                if term_idx == 0:
                    file_offset -= recon.offset_into_first_range
                
                # 从 all_chunks 中提取该 term 用到的 chunk
                segment = extract_segment_from_chunks(
                    all_chunks, term.chunks[0], term.chunks[1]
                )
                
                # 覆写
                f.seek(file_offset)
                f.write(segment)
                logger.info(f"已修复 term #{term_idx}, offset={file_offset}, len={len(segment)}")
    
    # 3. 重新验证文件级 SHA256
    logger.info("[FileVerifier] 修复完成，重新验证文件 SHA256...")
    final_sha256 = compute_sha256(output_path)
    if final_sha256 != expected_sha256:
        logger.error("[FileVerifier] 修复后文件 SHA256 仍然不匹配，放弃修复")
        return False
    
    logger.info("[FileVerifier] 修复成功！")
    return True
```

### 关键点

1. **不截断文件**：只覆写损坏的字节范围
2. **依赖 xorb 验证**：不依赖 per-term SHA256
3. **跨文件补全**：用 `repo_xorb_map` 获取完整 xorb 数据
4. **验证重新下载的数据**：确保修复后的 xorb 是正确的

### 位置

2. `chunk_assembler.py` 并行模式（~line 1159-1164）
3. `file_verifier.py` 修复（~line 413-418）

### 当前代码（以顺序模式为例）

```python
if end_chunk not in chunk_offsets_dict:
    sorted_chunks = sorted(chunk_offsets_dict.items())
    found = False
    for chunk_id, offset in sorted_chunks:
        if chunk_id >= end_chunk:
            end_byte = offset
            found = True
            break
    if not found:
        end_byte = xorb_data.data_size()
```

当 xorb 下载不完整时，`end_byte = data_size()` 会静默包含不该包含的数据。

### 方案

当 `end_chunk` 不在 dict 中且不是 xorb 末尾的下一个 chunk 时，报错：

```python
if end_chunk not in chunk_offsets_dict:
    # 检查是否是正常的 xorb 末尾（end_chunk == max_chunk_id + 1）
    max_chunk_id = max(chunk_offsets_dict.keys())
    if end_chunk == max_chunk_id + 1:
        end_byte = xorb_data.data_size()
    else:
        raise ValueError(
            f"[ChunkAssembler] Term #{term_idx}: end_chunk {end_chunk} "
            f"不在 xorb 中（max_chunk_id={max_chunk_id}），xorb 数据可能不完整"
        )
else:
    end_byte = chunk_offsets_dict[end_chunk]
```

**结论**：D5 依赖 D1 的 xorb_hash 验证。如果 D1 实现到位，xorb 数据不完整会在 xorb 验证阶段被发现，不会走到这个边界查找逻辑。因此 D5 可以推迟或作为防御性检查。
